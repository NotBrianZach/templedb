# Working with Deployed Projects

TempleDB includes commands to list and navigate to deployed projects. When you deploy a project, an `.envrc` file is automatically generated to make project CLIs available via direnv.

## Quick Start

### Deploy a Project

```bash
tdb deploy run bza
```

This will:
- Export the project from TempleDB
- Reconstruct files in `/tmp/templedb_deploy_<project>/working/`
- Generate an `.envrc` file for direnv integration
- Execute any deployment scripts (if configured)

## Commands

### List Deployed Projects

```bash
tdb deployed
```

Example output:
```
📦 Deployed Projects (3):

  ✓ bza
     Path: /tmp/templedb_deploy_bza/working
     Updated: 15 minutes ago

  ✓ woofs_projects
     Path: /tmp/templedb_deploy_woofs_projects/working
     Updated: 1 day ago

  ✓ system_config
     Path: /tmp/templedb_deploy_system_config/working
     Updated: 11 days ago
```

The ✓ indicates that an `.envrc` file exists for direnv integration.

### Navigate to Deployed Project

```bash
cd $(./templedb deploy path <project>)
# or
cd $(tdb deploy path <project>)
```

Example:
```bash
cd $(tdb deploy path bza)
```

Once you're in the directory, direnv will automatically:
- Load the Nix environment (if configured)
- Add project CLIs to your PATH
- Set up environment variables

### Optional: Create an Alias

Add to your `~/.bashrc` or `~/.zshrc` for convenience:

```bash
alias tcd='cd $(./templedb deploy path "$1" 2>/dev/null) 2>/dev/null || ./templedb deploy list'
```

Then use:
```bash
tcd bza
```

## How It Works

### Automatic .envrc Generation

When you deploy a project, TempleDB automatically generates an `.envrc` file in the deployed directory. This file:

1. **Detects and uses Nix environments** - If `flake.nix` or `shell.nix` exists, it will use them
2. **Adds project directory to PATH** - Makes project CLIs (like `bza.mjs`) available
3. **Makes scripts executable** - Automatically `chmod +x` common CLI files
4. **Exports environment variables** - Sets `<PROJECT>_PROJECT_ROOT`
5. **Loads .env files** - If a `.env` file exists, it's automatically loaded

Example generated `.envrc`:
```bash
# direnv configuration for bza (deployed)
# Auto-generated by TempleDB deployment

# Use Nix environment if available
if has nix_direnv_version || has nix; then
  if [ -f "flake.nix" ]; then
    use flake
  elif [ -f "shell.nix" ]; then
    use nix
  fi
fi

# Add current directory to PATH for project CLI
PATH_add "$(pwd)"

# Make project CLIs executable
for pattern in *.mjs *.py *.sh cli bza; do
  for file in $pattern; do
    if [ -f "$file" ] && [ ! -x "$file" ]; then
      chmod +x "$file"
    fi
  done
done

# Export project root
export BZA_PROJECT_ROOT="$(pwd)"

# Load environment variables from .env if present
if [ -f ".env" ]; then
  dotenv .env
fi

echo "✓ bza environment loaded (deployed)"
```

### Using direnv

When you navigate to a deployed project, direnv automatically loads the environment:

```bash
cd $(tdb deploy path bza)
direnv allow  # First time only
# Environment is now loaded, bza.mjs is in PATH
```

## Advanced Usage

### Deployment Workflow

1. **Initial deployment**:
   ```bash
   tdb deploy run myproject --dry-run  # Preview
   tdb deploy run myproject            # Execute
   ```

2. **Navigate to deployed project**:
   ```bash
   cd $(tdb deploy path myproject)
   ```

3. **Work with the deployed project** - All CLIs and environment variables are ready to use

4. **View deployment history**:
   ```bash
   tdb deploy history myproject
   ```

### Integration with Development

You can use the same workflow for both development and deployment:

1. **Development** (in original repo):
   ```bash
   cd /path/to/myproject
   # Use direnv with original .envrc
   ```

2. **Deployed version** (TempleDB managed):
   ```bash
   tdb cd myproject
   # Use direnv with auto-generated .envrc
   ```

## Troubleshooting

### direnv not loading automatically

Make sure direnv is installed and hooked into your shell. Add to your `~/.bashrc` or `~/.zshrc`:

```bash
# For bash
eval "$(direnv hook bash)"

# For zsh
eval "$(direnv hook zsh)"
```

### Project CLI not found after cd

Allow direnv the first time you enter a deployed project:
```bash
direnv allow
```

After that, it will load automatically whenever you cd into the directory.

### Old deployments taking up space

Deployed projects are stored in `/tmp/templedb_deploy_*/`. You can safely remove old ones:
```bash
rm -rf /tmp/templedb_deploy_<project>
```

Or clean all deployments:
```bash
rm -rf /tmp/templedb_deploy_*
```
