================================================================================
DATA FLOW: SINGLE INTERACTION
================================================================================

1. USER TYPES PROMPT
   └─> "Can you add authentication?"

2. CAPTURE SCRIPT DETECTS
   └─> Extracts: content, files mentioned, timestamp

3. POST TO VIBE SERVER
   └─> POST /api/vibe/interaction
       {
         "session_id": 123,
         "role": "user",
         "interaction_type": "user_prompt",
         "content": "Can you add authentication?"
       }

4. SERVER PROCESSES
   ├─> Extracts topics (authentication, feature)
   ├─> Detects files mentioned
   ├─> Assigns sequence number
   └─> Stores in vibe_claude_interactions

5. CLAUDE RESPONDS
   └─> "I'll create a JWT auth system..."

6. CAPTURE DETECTS RESPONSE
   ├─> Measures latency (e.g., 2.5s)
   ├─> Parses code blocks
   └─> Extracts tool uses

7. POST RESPONSE
   └─> POST /api/vibe/interaction (response)

8. SERVER PROCESSES RESPONSE
   ├─> Stores response in vibe_claude_interactions
   ├─> Extracts code snippets → vibe_interaction_code_snippets
   ├─> Creates pair → vibe_interaction_pairs
   └─> Updates stats → vibe_session_stats (via trigger)

9. DATA NOW QUERYABLE
   ├─> Full conversation in vibe_conversation_view
   ├─> Code snippets in vibe_code_generation_view
   ├─> Topics in vibe_topics_view
   └─> Stats in vibe_session_quality_view

================================================================================
QUERY FLOW: RETRIEVE CONTEXT
================================================================================

USER WANTS: "How did I implement authentication before?"

1. RUN QUERY COMMAND
   └─> tdb vibe-query search "authentication" --limit 10

2. CLI EXECUTES SQL
   └─> SELECT * FROM vibe_conversation_view
       WHERE content LIKE '%authentication%'
       ORDER BY created_at DESC LIMIT 10

3. DATABASE RETURNS RESULTS
   └─> 10 interactions about authentication

4. CLI FORMATS OUTPUT
   └─> Shows: session, timestamp, role, content preview
       Highlights: helpful interactions, commits

5. USER SELECTS SESSION
   └─> tdb vibe-query history 123 --format markdown

6. FULL CONVERSATION EXPORTED
   └─> Markdown file with complete conversation

7. USER PASTES INTO NEW SESSION
   └─> Claude Code gets context from past work!

================================================================================
ANALYTICS FLOW
================================================================================

QUESTION: "What tools do I use most?"

1. QUERY
   └─> SELECT tool_name, COUNT(*) as uses
       FROM vibe_claude_interactions
       WHERE tool_name IS NOT NULL
       GROUP BY tool_name
       ORDER BY uses DESC

2. RESULT
   ├─> Read: 234 uses
   ├─> Edit: 156 uses
   ├─> Bash: 89 uses
   └─> Grep: 67 uses

QUESTION: "Which sessions were most productive?"

1. QUERY (pre-built view)
   └─> SELECT * FROM vibe_session_quality_view
       ORDER BY productive_turns DESC

2. RESULT
   └─> Sessions ranked by commits, helpful responses

================================================================================
PERFORMANCE CHARACTERISTICS
================================================================================

WRITE PERFORMANCE:
• ~5-10ms to log single interaction (local SQLite)
• Async API, non-blocking
• WAL mode for concurrent access

READ PERFORMANCE:
• Indexed on session_id, sequence, created_at
• <50ms for session history (1000 interactions)
• <100ms for search across all sessions
• Views are pre-joined for fast queries

STORAGE:
• ~2KB per interaction (without embeddings)
• ~4KB per interaction (with 384-dim embeddings)
• 1000 interactions = ~2-4MB

SCALABILITY:
• Tested up to 100,000 interactions
• Pagination support for large result sets
• VACUUM recommended after large deletions
